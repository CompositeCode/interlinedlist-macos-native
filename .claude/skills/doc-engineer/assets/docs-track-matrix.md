# Documentation Track Matrix

Use this matrix to place content in the right documentation track.

## Engineering Docs
Audience: engineers and maintainers
Purpose: architecture, API integration, internals, design decisions
Typical content:
- system architecture
- service boundaries
- API request and response mapping
- error handling and retry strategy

## User Docs
Audience: application end users
Purpose: tasks, workflows, troubleshooting
Typical content:
- getting started
- key features and workflows
- troubleshooting guides
- known limitations and workarounds

## Repository Docs
Audience: contributors to this repository
Purpose: local setup, standards, contribution workflows
Typical content:
- local development setup
- test and lint commands
- PR and review process
- branching and release notes process

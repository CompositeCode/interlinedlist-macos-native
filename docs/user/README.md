# InterlinedList for macOS — User Documentation

**Audience:** application end users.

This folder is the user-track index for InterlinedList on macOS. The same content ships inside the app as the native Help Book (Help menu, or press Command + ?). The Markdown files here are the single source of truth; the Help Book HTML mirrors their wording.

## Contents

- [Getting started](getting-started.md) — sign in and a tour of the main window.
- [Feature status](feature-status.md) — what is shipped today versus coming in a future update, by milestone.

## Where to find more help

- In the app: **Help > InterlinedList Help** (or Command + ?).
- On the web: [interlinedlist.com](https://interlinedlist.com).

## Track scope

This folder contains only user-facing documentation — tasks, workflows, troubleshooting, and known limitations. Engineering documentation lives in [../](../) (api-coverage, decisions, progress, spikes). Repository / contributor documentation does not exist as a separate folder today; contributors should start with the top-level [PLAN.md](../../PLAN.md) and [ORCHESTRATION.md](../../ORCHESTRATION.md).
